# 依頼内容
現在運営しているWebサイトのUI/UXを、モダンかつ洗練されたデザインへ全面刷新します。
あなたは「UI/UXに精通したシニアフルスタックエンジニア」として、既存コードをベースにモダンなライブラリへ移行・再構築する指揮をとってください。
# 刷新のゴール
- デザインシステム：洗練、ミニマル、高級感。余白とタイポグラフィを重視。
- 技術スタック：Next.js (App Router), Tailwind CSS, Framer Motion, shadcn/ui, Aceternity UI。
- 目標：「動き」による没入感のある体験（スクロール演出、ホバーエフェクト等）。
# 開発プロセス（重要：一気に書き換えないこと）
以下の手順で進めます。まずは「フェーズ1」の分析と計画のみ回答してください。
### フェーズ1：現状分析と計画策定
1. 現在のファイル構成と主要コンポーネントを分析してください。
2. 今回の刷新で「流用すべきコンテンツ要素」と「新しく作り直すべきコンポーネント」を整理してください。
3. 既存のサイトを「壊さずに」、Tailwind CSSやshadcn/uiを導入するための最適な手順を提案してください。
4. 導入すべき共通レイアウト（Layout, Navbar, Footer）の骨子を提案してください。
### フェーズ2以降（フェーズ1完了後に順次実施）
- UIコンポーネントの逐次置換（shadcn/uiベースへの移行）
- Framer Motionを用いたアニメーション演出の実装
- コンテンツのレイアウト調整とレスポンシブ対応
まずは、フェーズ1の計画について回答をお願いします。

成功させるための「AI活用のコツ」
既存サイトを大幅に修正する場合、以下の点に注意してください。

プロジェクトフォルダ全体を確実に認識してください。現在のディレクトリ構造を知り、的外れなコードを生成しないようにしてください。
「shadcn/ui」はコンポーネント単位で追加する
一度に全ページを直そうとせず、**「まずはヘッダーだけ」「次はトップページのHeroセクションだけ」**と絞って指示を出してください。
例：「components/Hero.tsx をshadcn/uiとFramer Motionを使って、スクロールでふわっと現れるデザインに書き直して」といった具合です。
「アクセシビリティ」を忘れない
デザインを優先するあまり、読み込みが遅くなったり、スマホで見にくくなったりしがちです。「モバイルファーストで、かつ画面遷移がスムーズな設計にして」と付け加えると、パフォーマンスも意識してくれます。
